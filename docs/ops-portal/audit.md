# Audit Log Viewer

The Audit Log Viewer (`/ops/audit`) provides platform administrators with a unified, cross-tenant view of all audit entries on the platform, including both tenant-level events and `PLATFORM_OPS_*` platform administration events.

## Overview

The ops-level audit log aggregates entries from every tenant into a single searchable interface. This gives platform admins a complete picture of platform-wide activity without having to switch between individual tenant accounts.

:::info Tenant Audit Trail
Each business tenant also has their own audit trail accessible from the standard application. The Ops Audit Log Viewer is an additive, read-only view that includes everything the tenant sees **plus** all `PLATFORM_OPS_*` events generated by platform admin actions against that tenant's data.
:::

## Accessing the Audit Log Viewer

Navigate to **Ops Portal > Audit Log** or visit `/ops/audit` directly. 2FA must be active for the current session.

## Summary Cards

The top of the page shows four summary metrics:

| Card | Description |
|------|-------------|
| **Total Entries** | Total audit entries across all tenants |
| **Platform Ops Events** | Count of `PLATFORM_OPS_*` events in the selected period |
| **Critical Events** | High-severity events requiring review |
| **Chain Integrity** | Percentage of entries with a verified hash chain |

## Filtering

### By Scope

| Filter | Shows |
|--------|-------|
| **All** | Every entry across all tenants |
| **Platform Ops Only** | Only `PLATFORM_OPS_*` entries |
| **Tenant: [name]** | All entries for a specific tenant |

### By Event Type

Filter to specific event categories:

- Transaction events
- VAT return events
- User authentication events
- Security events
- Settings changes
- Platform ops events (`PLATFORM_OPS_*`)

### By Date Range

Set a **Start Date** and **End Date** to focus the log on a specific reporting period.

### By Severity

- **All** — Show every entry
- **Warning** — Warning and Critical only
- **Critical** — Critical entries only

## Audit Entry Table

| Column | Description |
|--------|-------------|
| **Timestamp** | UTC time the event was recorded |
| **Event Type** | Category badge — `PLATFORM_OPS_*` events display a distinct purple badge |
| **Action** | Human-readable description of what happened |
| **Actor** | User or system that performed the action |
| **Tenant** | Business tenant the event belongs to |
| **Severity** | Normal, Warning, or Critical |
| **IP Address** | Source IP address |
| **Chain** | ✓ Verified / ✗ Broken |

Click any row to expand the full audit entry, including the before/after data snapshot and the cryptographic hash values.

## Hash Chain Verification

The platform maintains a separate hash chain for `PLATFORM_OPS_*` events, independent of each tenant's own audit chain. Platform admins can verify both chains:

### Verify a Tenant Chain

1. Open the filter and select a specific tenant
2. Click **Verify Chain Integrity**
3. Review the results:
   - Total entries checked
   - Chain status (Intact / Broken)
   - Index of any broken links

### Verify the Platform Ops Chain

1. Set the scope filter to **Platform Ops Only**
2. Click **Verify Chain Integrity**
3. Review the results

:::warning Broken Chain
A broken chain link may indicate unauthorized modification of audit data. Escalate to the security team immediately and do not make further changes to the affected tenant's data until the investigation is complete.
:::

## Exporting Audit Data

Platform admins can export filtered audit data for compliance reviews, legal requests, or external archival:

1. Apply the desired filters (tenant, date range, event type, severity)
2. Click **Export**
3. Choose the format: **CSV** or **JSON**
4. The download begins immediately

Exports include all visible columns plus the raw hash values for chain verification purposes. A `PLATFORM_OPS_AUDIT_EXPORTED` event is recorded with the export parameters.
